Superstore Analysis

Overview:

Since 2012, our client Superstore has implemented a robust infrastructure for collecting data on sales, customers, product performance, and more.

However, while the data is regularly collected and stored in a data lake, the Superstore team has yet to implement a dashboard or sophisticated reporting process. As a result, they have to resort manually pulling data from various sources and create time-intensive and non-scaleable Excel reports. This lag of real-time analysis leads to “gut feeling” types of work/management styles and an overall blurred sense of company-wide performance.

The CEO and other high-level executives have a number of questions on the performance of their company, but cannot seem to get their team to provide fast enough insights, so they decided to get help from BACC to unlock their data.

Business Requirements:
  1. How do we fix our data process so reports get out faster?
  2. How are sales in each region year-over-year?
  3. Which products are performing well? Which are not?
  4. Who are our most profitable customers?
  5. How efficient is the warehouse team at shipping products on time?
  6. What does our sales projection look like for next year?

Data Pipeline Revamp

Original Pipeline


During this process data were extracted mainly from sources like Salesforece, SAP, and Oracle in the form of excel files. The data was then joined and cleaned in Excel into a master dataset for exploratory analysis and visualization using pivot table and pivot charts.

There are many downsides to this process. The entire ETL process is done using local computer memory, Superstore team has to work with super large excel file, and they would need to go through the entire process every time for a new analysis.

Optimized Pipeline Process


We created a solution by incorporating Relational Database Management and cloud computing. Instead of extracting data onto the local computer, we stored them on a database on the cloud that would provide us with limitless storage capacity.

They are stored in a relational database structure, making our data more scalable and easy to maintain. The data is then loaded onto a cloud platform for cleaning and transformation. Utilizing cloud computing can drastically reduce the error in the ETL process made through concatenating different excel files. In addition, cloud platform also allows EDA and aggregation, so we can feed the most accurate and relevant data to our Tableau dashboard.The size of data can be stored is increased from local storage capacity, which is typically less than 500GB, to over 500 TB. The work constraint is lifted by switching to cloud as well since performing ETL on local computer can easily eat up computer memory.

Traditional Data Pipeline Vs. Cloud

This table shows some of the improvement after adapting to the new process. The storage is no longer on local computer that takes computer memory. The size of data can be stored is increased from local storage capacity, which is typically less than 500GB, to over 500 TB. The work constraint is lifted by switching to cloud as well since performing ETL on local computer can easily eat up computer memory.

By utilizing Cloud and RDS, we greatly improved the data pipeline’s work efficiency and performance, mitigated work constraints, and created more structured data with much higher clarity and accuracy.

  • Second class can also use some attention to decrease late shipments.
  • Standard shipping is doing pretty decent with only 10% of the items being shipped late
